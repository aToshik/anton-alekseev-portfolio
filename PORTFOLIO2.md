### Кейс 2 — Design System for AI Agents
**Нарратив:** Design System built for AI — уникальная история на пересечении DS и agentic development

**Теглайн:** How I evolved a design system to work with AI coding agents — and what I learned about context, documentation, and the future of design handoff.

**Структура кейса:**
1. **Context** — Kissterra переходит на AI-agent-driven development (Cursor, Claude Code), команда из 4 frontend-инженеров
2. **Problem** — AI-агент не понимает нюансы компонентов. Он не опытный разработчик — ему нужен явный контекст. Без него: неправильное использование компонентов, регрессии, длинные clarification-циклы
3. **Insight** — дизайн-система существует для людей, но AI читает её иначе. Нужен отдельный "слой" для машины
4. **Solution** — context-layer:
   - Embedded annotation docs прямо в Figma-компоненты (что делает, когда использовать, когда не использовать, edge cases)
   - component-doc.md в репо — всегда актуальны, всегда доступны агенту
   - Специализированные skills (Figma ↔ repo sync, diff detection, doc writing)
5. **Impact** — значительное снижение clarification-циклов, повышение качества генерации компонентов и layouts
6. **Reflection** — это не просто документация, это новый слой дизайн-системы. AX начинается с файлов Figma

**Визуальные ассеты:** скриншоты Figma с embedded docs, пример component-doc.md, before/after качества генерации (если есть скрины)

### Кейс 3 — AI Data Assistant (Beacon Chatbot)
**Нарратив:** AI Product Design — проектирование conversational + agentic UX

**Теглайн:** Designing a conversational co-pilot for insurance media buyers — trust, explainability, and the path to autonomous action.

**Структура кейса:**
1. **Context** — media buyers тратят часы на ручной анализ дашбордов. Цель: instant insights через natural language
2. **Problem** — conversational UI для сложных бизнес-данных: как сделать ответы понятными, trustworthy и actionable? Как проектировать, когда агент может ошибаться?
3. **Design challenges:**
   - Trust & explainability: почему агент даёт такую рекомендацию?
   - Human-in-the-loop: как показать рекомендацию так, чтобы пользователь мог принять/отклонить осознанно
   - Progressive autonomy: от "suggest" к "execute with approval"
   - Failure states: что показывать, когда агент не уверен
4. **Solution** — UI паттерны для conversational analytics: structured responses, confidence indicators, inline actions, approval flows
5. **Roadmap thinking** — как дизайн закладывает фундамент для полноценного агента
6. **Learnings** — чем отличается проектирование AI-продуктов от обычного SaaS

**Визуальные ассеты:** UI чатбота, flow рекомендации → approve → execute, error/uncertainty states
